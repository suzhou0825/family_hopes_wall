# 家庭许愿墙 MVP 产品方案

## 产品定位

家庭许愿墙用于加强父母和子女之间的沟通。孩子可以表达愿望，父母可以发布任务并设置奖励，孩子通过完成任务推动愿望点亮和兑现。

## 目标用户

- 父母：发布任务、设置奖励、审核完成、兑现愿望。
- 孩子：提交愿望、申领任务、提交完成、指定兑现人。

## MVP 范围

- 先做 Web 应用，兼容手机、iPad 和桌面浏览器。
- 先使用前端模拟数据跑通流程。
- 已接入 Supabase 登录和数据库保存。
- 打开网页后必须先登录账号密码，未登录只能看到登录/注册页。
- 账号资料保存到 Supabase `profiles` 表。
- Supabase 登录支持邮箱密码，并在前端提供 GitHub 登录入口。
- 当前 Supabase 第一版使用 `app_state` JSON 表保存完整应用数据。
- 当前本地已配置 Supabase 项目 `jqlgnsatgvwjadbzghfr`。
- 当前 Supabase 项目已执行 `supabase/schema.sql`。
- 未登录或未配置 Supabase 时，使用浏览器 localStorage 保存本机演示数据。
- 后续需要继续拆分数据库表，并完善家庭级共享和权限规则。
- 不做苹果安装包、不做支付、不做商城、不做物流。
- 使用说明见 `docs/USER_GUIDE.md`。
- 开发约束见 `docs/DEVELOPMENT_CONSTRAINTS.md`。
- 外网访问部署说明见 `docs/DEPLOYMENT.md`。

## 核心功能

### 许愿

- 孩子可以提交愿望。
- 愿望类型包括物质奖励、旅游奖励、陪玩奖励等。
- 愿望需要支持具体描述，方便孩子说明想要什么、希望怎么实现。
- 愿望可以填写期望时间。
- 孩子可以指定希望由爸爸或妈妈兑现。
- 愿望状态：
  - 待申领：默认状态。
  - 兑换中：愿望已被父母关联为任务奖励。
  - 兑现中：关联任务已完成并通过父母审核。

### 任务

- 父母可以发布任务。
- 任务类型包括打卡任务、一次性任务、承诺任务。
- 每个任务必须设置奖励。
- 奖励可以是普通奖励，也可以关联孩子的某个愿望。
- 任务奖励需要选择奖励类型，奖励类型与愿望类型保持一致。
- 任务奖励需要填写具体奖励描述。
- 打卡任务必须设置打卡频率和截止时间。
- 打卡频率按周一到周日由父母勾选。
- 孩子可以每天在任务上打卡，任务看板展示完成次数、目标次数和截止时间。
- 任务状态：
  - 待申领：发布后尚未被孩子申领。
  - 完成中：孩子已申领并正在完成。
  - 已完成：孩子提交完成，并由父母审核通过。

### 审核与兑现

- 孩子提交任务完成后，需要父母审核。
- 审核通过后，任务进入已完成。
- 如果任务关联了愿望，愿望进入兑现中。
- 兑现本身由家庭线下完成，系统先记录状态。

## 页面规划

- 首页许愿墙：展示所有孩子的愿望进展、关联任务和父母任务状态。
- 孩子登录后，可以在许愿墙上主动申领待申领任务。
- 孩子许愿页：孩子创建愿望、查看自己的愿望。
- 孩子我的任务页：展示自己申领的任务、任务状态和完成进展。
- 父母任务页：父母发布任务、关联愿望、审核任务。
- 家庭管理页：父母管理员添加家庭成员、设置角色、父母身份和孩子信息。
- 添加或编辑孩子时，需要设置孩子性别和家庭称呼，称呼可选哥哥、弟弟、姐姐、妹妹。
- 添加或编辑父母时，设置爸爸或妈妈身份。

## 账号与权限

- 应用登录由 Supabase Auth 提供，支持邮箱密码和 GitHub OAuth。
- 账号管理支持保存昵称和修改密码。
- 父母账号可以访问任务发布和家庭管理。
- 孩子账号可以访问许愿板块、我的任务板块，并在许愿墙申领任务。
- 父母账号不显示孩子许愿入口。
- 孩子账号不显示任务发布入口。
- 家庭成员由管理员手动添加。
- Supabase 登录用于云端保存数据；页面右上角的家庭成员切换用于模拟父母/孩子视角。
