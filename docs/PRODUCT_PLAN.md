# 家庭许愿墙 MVP 产品方案

## 产品定位

家庭许愿墙用于加强父母和子女之间的沟通。孩子可以表达愿望，父母可以发布任务并设置奖励，孩子通过完成任务推动愿望点亮和兑现。

## 目标用户

- 父母：发布任务、设置奖励、审核完成、兑现愿望。
- 孩子：提交愿望、申领任务、提交完成、指定兑现人。

## MVP 范围

- 先做 Web 应用，兼容手机、iPad 和桌面浏览器。
- 先使用前端模拟数据跑通流程。
- 已接入 Supabase 数据库保存。
- 登录使用本应用自有账号体系，不使用 Supabase Auth 作为家庭成员登录。
- 打开网页后必须先登录账号密码，未登录只能看到登录/注册页。
- 账号资料保存到 Supabase `app_accounts` 表。
- 注册只支持普通账号。
- 主动注册只能创建父母账号，注册时选择爸爸或妈妈。
- 主动注册会创建新家庭；第二个父母账号和孩子账号由已登录父母在家庭管理中创建，加入同一个家庭。
- 账号只能包含小写字母、数字、下划线，长度 3 到 32 位。
- 注册需要确认密码，并校验密码至少 8 位且包含字母和数字。
- 密码输入框支持显示/隐藏。
- 当前 Supabase 第一版使用 `app_family_data` JSON 表保存完整家庭应用数据。
- Supabase 在四张账号与家庭核心表之外，新增积分账户、积分流水、宠物领养和宠物互动四张业务表。
- 当前本地已配置 Supabase 项目 `jqlgnsatgvwjadbzghfr`。
- 当前代码要求 Supabase 项目重新执行最新版 `supabase/schema.sql`，以创建积分和宠物业务表及 RPC。
- 未配置 Supabase 时，使用浏览器 localStorage 保存本机演示数据。
- 后续需要继续拆分数据库表，并完善家庭级共享和权限规则。
- 本地调试、GitHub/Vercel 线上版本必须连接同一个 Supabase 项目，避免本地和线上数据不一致。
- 不做苹果安装包、不做支付、不做商城、不做物流。

- 开发约束见 `docs/DEVELOPMENT_CONSTRAINTS.md`。
- 外网访问部署说明见 `docs/DEPLOYMENT.md`。

## 核心功能

### 许愿

- 孩子可以提交愿望。
- 愿望类型包括物质奖励、旅游奖励、陪玩奖励、积分奖励和其他愿望。
- 愿望保存精确提交时间，并在许愿墙展示。
- 新愿望或编辑愿望不得与未归档愿望重名。
- 愿望需要支持具体描述，方便孩子说明想要什么、希望怎么实现。
- 愿望可以填写期望时间。
- 孩子可以指定希望由爸爸或妈妈兑现。
- 愿望状态：
  - 待申领：默认状态。
  - 兑换中：愿望已被父母关联为任务奖励。
  - 兑现中：关联任务已完成并通过父母审核。
  - 已兑换：父母确认愿望已兑现，愿望从许愿墙归档。

### 任务

- 父母可以发布任务。
- 任务类型包括打卡任务、一次性任务、承诺任务。
- 每个任务必须设置奖励。
- 奖励可以是普通奖励，也可以关联孩子的某个愿望。
- 任务奖励需要选择奖励类型，奖励类型与愿望类型保持一致。
- 任务奖励需要填写具体奖励描述。
- 任务奖励类型增加积分奖励，并记录积分数量。
- 任务保存精确提交时间，并在任务公示中展示。
- 父母发布任务、孩子申报任务不得与未完成任务重名。
- 孩子申报任务时可以选择奖励类型；积分奖励填写建议积分。
- 父母可以一键复制已公示任务，副本自动生成不重复名称并重置领取和执行状态。
- 打卡任务必须设置打卡频率和截止时间。
- 打卡频率按周一到周日由父母勾选。
- 孩子可以每天在任务上打卡，任务看板展示完成次数、目标次数和截止时间。
- 任务状态：
  - 待申领：发布后尚未被孩子申领。
  - 完成中：孩子已申领并正在完成。
  - 已完成：孩子提交完成，并由父母审核通过。
- 已完成任务从许愿墙任务看板归档；孩子仍可在“我的任务”查看，父母可在任务管理的历史任务中查看。

### 审核与兑现

- 孩子提交任务完成后，需要父母审核。
- 审核通过后，任务进入已完成。
- 如果任务关联了愿望，愿望进入兑现中。
- 兑现本身由家庭线下完成，系统先记录状态。

## 页面规划

- 登录后首页采用三栏入口布局：许愿墙、任务公示、积分区。
- 首页不显示独立导航条，点击整张卡片进入对应专属页面。
- 许愿墙卡片动态展示数据库中未完成愿望，任务公示卡片动态展示已审批且未完成任务。
- 首页愿望和任务滚动条目展示精确提交时间，帮助家庭成员区分同类内容。
- 许愿墙和任务公示窗口一次展示约 4 条内容，其余数据滚动展示。
- 许愿墙和任务公示根据登录账号权限显示发布愿望或发布任务按钮。
- 发布愿望、发布任务或申报任务按钮位于首页卡片头部，不放在卡片底部。
- 顶部展示动画头像、账号昵称、账号、家庭角色，并提供家庭管理、账号管理和退出入口。
- 顶部不展示“已保存到 Supabase”等内部保存状态文字。
- 首页许愿墙：展示所有孩子的愿望进展、关联任务和父母任务状态。
- 孩子登录后，可以在许愿墙上主动申领待申领任务。
- 孩子许愿页：孩子创建愿望、查看自己的愿望。
- 孩子我的任务页：展示自己申领的任务、任务状态和完成进展。
- 我的任务同时展示任务提交时间、孩子完成提交时间和父母审核后的最终完成时间。
- 父母任务页：父母发布任务、关联愿望、审核任务。
- 家庭管理页：父母添加父母账号或孩子账号，设置初始密码、身份和孩子信息。
- 家庭管理页删除成员时，需要同步删除该成员登录账号和会话。
- 添加或编辑孩子时，需要设置孩子性别和家庭称呼，称呼可选哥哥、弟弟、姐姐、妹妹。
- 父母身份在主动注册时设置为爸爸或妈妈。
- 登录页和应用整体采用粉紫、暖粉、奶油色为主的柔和圆角卡片风格，并使用原创家庭主题插画。
- 积分中心已接入真实数据库：父母查看孩子排行和滚动宠物陈列，孩子查看真实余额、流水与兑换。
- 第一版宠物库提供 2 只狗和 2 只猫；仅孩子可领取且每人最多 2 只，领取关系保存到家庭数据。
- 孩子首页积分卡提供大型宠物陈列区，展示宠物动作、领养时间、成长值、心情和当天心理感悟；两只宠物自动轮播。
- 宠物成长按领养天数累计，每天增加 10 点；每日心情、感悟、互动状态和记录保存到数据库。
- 每个孩子首次建立积分账户时由系统授予 100 成长星；任务审核、兑换、领养和弃养均记录真实流水。
- 第一只宠物免费，第二只 500 成长星，弃养 2000 成长星。
- 孩子愿望和任务申报的成长星奖励上限为 100；父母发布任务支持非积分奖励与成长星组合。
- 宠物专属页支持喂养、玩耍、装扮和弃养；拍照分享、串门后续开发。
- 虚拟社区第一阶段静态展示虚拟宠物、虚拟装扮、虚拟装修和虚拟店面，后续再接入业务数据。

## 账号与权限

- 应用登录由本应用账号体系提供，登录时通过 Supabase RPC 校验数据库中的账号密码哈希。
- 只支持普通账号注册登录，不使用邮箱或 GitHub 登录。
- 主动注册只能创建父母账号。
- 第二个父母账号和孩子账号只能由已登录父母在家庭管理中创建，才能共享同一个家庭数据。
- 账号管理支持保存昵称和修改密码。
- 账号管理支持选择内置动画头像；头像编号保存到 `app_accounts.avatar_id`。
- 头像配置库包含爸爸、妈妈、哥哥、弟弟、姐姐、妹妹六类原创人物头像。
- 家庭管理的成员卡片展示成员头像，旧数据按家庭角色回退到默认头像。
- 账号修改密码必须输入两次并校验一致。
- 孩子可以申报任务；父母审批通过后任务进入公示，孩子可自行申领。
- 父母账号可以访问任务发布和家庭管理。
- 孩子账号可以访问许愿板块、我的任务板块，并在许愿墙申领任务。
- 父母账号不显示孩子许愿入口。
- 孩子账号不显示任务发布入口。
- 家庭成员由父母手动添加。
- Supabase 数据库用于云端保存账号、会话和家庭数据；页面右上角只读展示当前登录账号绑定的家庭成员和具体角色，不提供角色切换。
