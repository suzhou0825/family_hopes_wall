# 家庭许愿墙使用说明

## 项目交接文档

- 当前交接文档：`handover/PROJECT_HANDOVER.md`。
- 可重复执行的生成规范：`PROJECT_HANDOVER_GENERATION_SPEC.md`。

- 本地访问地址：`http://localhost:3333`

- 外网部署说明见 `docs/DEPLOYMENT.md`。

## Supabase 配置

当前已接入 Supabase 数据库保存。登录使用本应用自己的账号体系，不使用 Supabase Auth 作为家庭成员登录账号。

配置步骤：

- 在 Supabase 创建项目。
- 在 Supabase SQL Editor 执行 `supabase/schema.sql`。
- 复制 `.env.example` 为 `.env.local`，或确认 `.env.local` 已存在。
- 填写：
  - `NEXT_PUBLIC_SUPABASE_URL`
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- 重启本地服务。

不要把 Supabase `service_role` key 写进前端或发给别人。
当前本地已配置 Supabase 项目 `jqlgnsatgvwjadbzghfr`。
当前 Supabase 需要执行最新的 `supabase/schema.sql`，创建 `app_families`、`app_accounts`、`app_sessions`、`app_family_data` 和 RPC 函数。
执行最新脚本后，旧版本遗留的 `profiles`、`app_state`、`app_data` 会被删除。
如果页面一直停在“正在检查登录状态”，优先检查 Vercel 或本地环境变量是否配置了 `NEXT_PUBLIC_SUPABASE_URL` 和 `NEXT_PUBLIC_SUPABASE_ANON_KEY`。

## 数据保存

- 登录应用账号后，数据通过 Supabase RPC 保存到 `app_family_data` 表。
- 账号密码保存在 `app_accounts` 表，其中密码只保存哈希。
- 登录会话保存在 `app_sessions` 表，浏览器本地只保存会话 token。
- 未配置 Supabase 时，数据保存到浏览器 `localStorage`，仅用于开发兜底。
- 当前第一版云端数据以 JSON 形式保存，后续会拆成家庭成员、愿望、任务、打卡记录等独立数据表。
- 本地调试和 Vercel 线上环境使用同一个 Supabase 项目时，登录同一账号可以看到同一份云端数据。

## 当前账号角色

打开网页后必须先登录应用账号。登录后页面右上角只显示该账号绑定的家庭成员和具体角色，不能切换成其他家庭成员。

注册规则：

- 只使用普通账号注册，不使用邮箱登录。
- 主动注册只能注册父母账号。
- 注册时必须选择父母身份：爸爸或妈妈。
- 主动注册会创建一个新家庭。
- 第二个父母账号应由已登录父母在家庭管理中创建，不能通过主动注册加入同一个家庭。
- 账号只能包含小写字母、数字、下划线，长度 3 到 32 位。
- 注册时必须输入两次密码。
- 两次密码必须一致。
- 密码至少 8 位。
- 密码必须同时包含字母和数字。
- 字母区分大小写。
- 登录、注册和修改密码的密码框支持显示/隐藏密码。
- 父母创建孩子账号时，初始密码和确认初始密码也支持显示/隐藏。

## 账号管理

登录后页面上方显示账号管理。

支持：

- 查看当前登录账号
- 查看或保存账号昵称
- 编辑账号昵称
- 修改密码
- 确认新密码
- 选择内置动画头像
- 退出登录

账号名、账号昵称和头像编号保存到 Supabase `app_accounts` 表。密码只保存哈希，不在前端保存明文。

修改密码时必须输入两次相同的新密码；不修改密码时两个密码框都可以留空。

头像库包含爸爸、妈妈、哥哥、弟弟、姐姐和妹妹六类人物形象，人物图由 CSS 提供轻量浮动动画。

## 登录后首页

- 左侧许愿墙：查看愿望摘要并进入许愿墙；孩子账号显示发布愿望按钮。
- 中间任务公示：查看任务摘要并进入任务中心；父母账号显示发布任务按钮。
- 右侧积分区：查看静态积分余额、兑换物滚动展示和虚拟社区预览。
- 首页不显示单独导航条，点击整张卡片进入对应页面。
- 三张卡片标题靠左展示，不显示数字序号。
- 许愿墙和任务公示卡片会动态滚动展示当前未完成数据，发布或申报入口位于卡片头部。
- 许愿墙和任务公示的说明摘要位于卡片底部上方，避免遮挡动态内容。

## 积分中心

- 当前积分中心是静态原型，不会真实增加或扣除积分。
- 页面展示积分累计和消耗示例、实物兑换、虚拟物品兑换及虚拟社区。
- 虚拟社区暂时展示虚拟宠物、虚拟装扮、虚拟装修和虚拟店面规划。

- 父母账号：可以访问任务发布和家庭管理。
- 孩子账号：可以访问许愿板块、我的任务，并在许愿墙申领任务。
- 父母账号和孩子账号都可以由已登录父母在家庭管理中创建，用于加入同一个家庭。

## 孩子使用流程

### 发布愿望

进入许愿板块后，孩子可以发布或编辑愿望。

需要填写：

- 愿望名称
- 愿望描述
- 愿望类型：物质奖励、旅游奖励、陪玩奖励、积分奖励、其他愿望
- 希望谁兑现：爸爸或妈妈
- 期望时间
- 积分奖励需要填写积分数量。
- 未归档愿望不能使用完全相同的名称。
- 许愿墙会展示愿望提交时间。

### 申领任务

孩子可以在许愿墙上申领待申领任务，也可以进入我的任务查看可申领任务。

### 申报任务

- 孩子在任务中心填写任务名称、类型、说明和奖励类型后提交申报。
- 选择积分奖励时填写建议积分，其他奖励填写建议奖励说明。
- 申报名称不能与现有未完成任务完全一致。
- 申报任务先进入待父母审批，不会直接出现在任务公示中。
- 父母在任务中心审批通过后，任务进入公示和可申领列表。
- 孩子可以自行领取审批通过的任务。

### 完成任务

- 打卡任务：孩子每天按任务频率进行打卡。
- 一次性任务、承诺任务：孩子完成后提交完成。
- 任务完成后需要等待父母审核。

### 查看我的任务

我的任务展示孩子自己申领的任务、状态和进展。

## 父母使用流程

### 发布任务

进入任务发布后，父母可以发布或编辑任务。

需要填写：

- 任务名称
- 任务类型：打卡任务、一次性任务、承诺任务
- 奖励类型：物质奖励、旅游奖励、陪玩奖励、积分奖励、其他愿望
- 奖励描述
- 可选关联孩子愿望
- 积分奖励需要填写积分数量。
- 发布或编辑任务时，名称不能与未完成任务完全一致。
- 任务公示会展示提交时间。

### 复制任务

- 父母可以在任务公示卡片点击复制同样任务。
- 副本保留任务类型、奖励和打卡频率，自动生成副本名称。
- 副本不会保留领取人、打卡、提交状态和关联愿望；打卡截止时间按原任务周期顺延。

如果选择打卡任务，还需要设置：

- 打卡频率：周一到周日可多选
- 截止时间

### 审核任务

孩子完成任务后，父母在任务发布页审核。

- 审核通过后，任务进入已完成。
- 如果任务关联了愿望，愿望进入兑现中。
- 已完成任务会从许愿墙任务看板归档，仍可在孩子的“我的任务”和父母任务页的历史任务中查看。

### 确认愿望已兑现

- 关联任务审核通过后，愿望进入兑现中。
- 父母在许愿墙点击“确认已兑现”，愿望进入已兑换状态并从许愿墙归档。
- 孩子可在许愿板块展开“历史愿望”查看已兑换记录。

### 家庭管理

父母可以添加父母账号、添加孩子账号，或编辑家庭成员资料。

家庭成员卡片会展示人物头像；旧成员没有头像编号时按爸爸、妈妈、哥哥、弟弟、姐姐或妹妹显示默认头像。

添加父母账号时：

- 设置父母姓名
- 设置登录账号
- 设置初始密码并确认密码
- 选择父母身份：爸爸或妈妈

添加孩子账号时：

- 设置孩子姓名
- 设置孩子登录账号
- 设置孩子初始密码并确认密码
- 选择性别：男孩或女孩
- 选择家庭称呼：哥哥、弟弟、姐姐、妹妹

编辑孩子资料时：

- 设置姓名
- 选择性别：男孩或女孩
- 选择家庭称呼：哥哥、弟弟、姐姐、妹妹

删除家庭成员时：

- 会同步删除该成员的登录账号和会话。
- 不能删除当前正在登录的账号。
- 删除孩子会同步移除孩子的愿望，并把孩子申领中的任务恢复为待申领。

## 状态说明

愿望状态：

- 待申领：愿望未关联任务。
- 兑换中：愿望已被任务关联。
- 兑现中：关联任务已完成，等待父母兑现。
- 已兑换：父母已确认兑现，愿望进入历史记录。

任务状态：

- 待申领：任务还没有孩子申领。
- 完成中：孩子已申领并正在完成。
- 已完成：父母已审核通过。

## 界面风格

- 登录页使用原创四人家庭插画和粉紫色柔和卡片设计。
- 登录方式仍为应用普通账号，不提供邮箱、GitHub 或其他第三方登录。

当前是本地原型，不要录入孩子真实姓名、学校、地址、联系方式等敏感信息。
